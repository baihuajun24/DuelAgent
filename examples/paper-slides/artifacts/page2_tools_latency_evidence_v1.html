Evidence: Tools Dominate Agent Latency / 证据：工具调用是 Agent 时延大头

A cleaner motivation slide focused on measured latency breakdowns from prior work.

Takeaway / 核心结论

  • Tool execution is often a major part of total latency.在真实 agent workload 里，工具执行经常占总时延的很大一部分，而不是只有 LLM 在慢。
  • Cold-start or init is not the main issue.真正的大头通常是工具执行本身，而不是初始化；只做 init 优化不够。
  • This supports speculative tool execution.如果能把工具执行和模型思考重叠起来，就能抓住最关键的优化空间。
